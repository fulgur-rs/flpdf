# flpdf-9hc.37: overlay/underlay source open warnings → CLI stderr

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to
> implement this plan task-by-task.

**Goal:** Print each overlay/underlay source PDF's
`repair_diagnostics()` to stderr in qpdf's format so users see the same
warnings they'd get from `qpdf --overlay ... --`. Exit code is
unchanged (qpdf 11.9.0 empirical: overlay/underlay source warnings do
not drive exit 3).

**Architecture:** Extract a small helper `emit_open_warnings(path, pdf)`
that reuses the existing `diagnostic_location` formatter. Call it from
`open_pdf_impl` (replacing the inline loop) and from
`build_overlay_specs` after each source open. No PDF byte-output
change; wiring-level tests only.

**Tech Stack:** Rust, `flpdf-cli` crate, `assert_cmd` + `predicates`
for CLI tests.

**Design:** See beads issue `flpdf-9hc.37` (`bd show flpdf-9hc.37`) for
full design incl. empirical qpdf verification.

/// printed to stderr in qpdf's `WARNING: <path>[ (offset N)]: <msg>`

**Step 1:** Add helper near the existing `diagnostic_location`
(around main.rs:4962), immediately below it:

```rust
/// Emit each of `pdf.repair_diagnostics()` to stderr in qpdf's shape
/// `WARNING: <file>[ (offset N)]: <message>`. Shared between the
/// primary-input open path (`open_pdf_impl`) and the overlay/underlay
/// source open path (`build_overlay_specs`), so both surfaces print
/// warnings identically.
fn emit_open_warnings<R: std::io::Read + std::io::Seek>(
    path: &Path,
    pdf: &Pdf<R>,
) {
    for diagnostic in pdf.repair_diagnostics().entries() {
        let location = diagnostic_location(path, diagnostic.offset);
        eprintln!("WARNING: {location}: {}", diagnostic.message);
    }
}
```

/// Regression guard: an overlay source with no `repair_diagnostics`
/// must not add any WARNING line — the emit helper must be silent on
/// a clean source.
#[test]
fn overlay_clean_source_emits_no_warnings() {
    let tmp = tempfile::tempdir().unwrap();
    let out = tmp.path().join("out.pdf");
    let clean = fixture("dangling-body-one-page.pdf");

    let mut cmd = Command::cargo_bin("flpdf").unwrap();
    cmd.args([
        "--overlay",
        &clean,
        "--",
        &clean,
        out.to_str().unwrap(),
    ])
    .assert()
    .success()
    .stderr(predicate::str::contains("WARNING:").not());
}

/// Two corrupt overlay sources → warnings for both sets are emitted
/// (this asserts the count / iteration wiring, not exact ordering).
#[test]
fn two_corrupt_overlay_sources_each_emit_warnings() {
    let tmp = tempfile::tempdir().unwrap();
    let a = tmp.path().join("a.pdf");
    let b = tmp.path().join("b.pdf");
    std::fs::write(&a, corrupt_xref_with_info_pdf()).unwrap();
    std::fs::write(&b, corrupt_xref_with_info_pdf()).unwrap();
    let out = tmp.path().join("out.pdf");

    let mut cmd = Command::cargo_bin("flpdf").unwrap();
    cmd.args([
        "--repair",
        "--overlay", a.to_str().unwrap(), "--",
        "--overlay", b.to_str().unwrap(), "--",
        &fixture("dangling-body-one-page.pdf"),
        out.to_str().unwrap(),
    ])
    .assert()
    .success()
    .stderr(predicate::str::contains(format!(
        "WARNING: {}: file is damaged",
        a.display()
    )))
    .stderr(predicate::str::contains(format!(
        "WARNING: {}: file is damaged",
        b.display()
    )));
}
