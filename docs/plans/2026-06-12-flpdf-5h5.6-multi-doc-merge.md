# Multi-Document Merge Primitive Implementation Plan (flpdf-5h5.6)

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (or subagent-driven-development) to implement this plan task-by-task.

**Goal:** `merge_documents(inputs: &mut [MergeInput])` — N 個のソース文書から選択ページを1つの fresh 文書へコピーしてマージ（qpdf `--pages` parity）。共有リソース dedup、AcroForm フィールド名衝突を `+N` でリネーム、document-level 情報は primary input のみ継承。

**Architecture:** fresh 空ターゲットに、各入力の選択ページ closure を**入力ごとに** `copy_objects` でコピー（`target.max()+1` 採番で disjoint）。**除去ページ null プレースホルダ機構（案 Y）**: 各入力で、選択ページの annotation dest と（primary のみ）outline/names dest が参照する**除去ページ ref も closure に含めて** copy する。`copy_objects` の単射 rewrite が dest を新 ref に変換（`[null]` inline 化を回避）し、コピー後にその除去ページ本体を `null` に置換する。これで qpdf `--pages` の null-out 忠実性（dest ref 保持・ページ null）を満たしつつ、**outline/annotation の追加 remap pass が不要になり** Step3/Step4 が共有 `/GoTo` を二度触る **ygoj ハザードが構造的に発生しない**。document-level（outline/names/openaction）オブジェクトは primary のみ closure に含める。AcroForm は primary の `/AcroForm` をベースに各入力のコピー済みページ widget の top-level field をマージ、`/T` 衝突を `<name>+<N>` でユニーク化。

**重要（extract との機構差）:** `extract_pages` は除去ページを closure に含めず annotation `/Dest` を **drop** する（qpdf "single addPage" パターン、document-level 継承なし）。merge は qpdf `--pages` 相当で document-level を継承するため、**除去ページを null プレースホルダで残し dest ref を保持**する（`outline_dest_remap.rs:22-40` が文書化した qpdf 挙動）。よって merge は `neutralize_absent_dests`（drop）を**使わない** — closure 拡張 + 除去ページ null 化が merge 専用機構。

**Tech Stack:** Rust, flpdf 内部 API（`copy_objects`, `page_object_closure`, `splice_pages`, `neutralize_absent_dests`, `remap_outline_and_dests` のヘルパー群, `AcroFormDocumentHelper::copy_fields_from`, `name_number_tree`）。

**Design:** beads issue `flpdf-5h5.6` の design フィールド（`bd show flpdf-5h5.6`）。

**作業ディレクトリ:** `/home/ubuntu/flpdf/.worktrees/flpdf-5h5-6-multi-doc-merge`（branch `feature/flpdf-5h5.6-multi-doc-merge`）

**コーディング規約:** `.claude/rules/pdf-rust-review-patterns.md`（不要 clone 禁止・間接参照 resolve・符号なしキャスト検証・走査境界）と `.claude/rules/pdf-rust-doc-review-patterns.md`（公開 doc に issue ID 禁止・英語のみ・`# Errors` 必須・intra-doc リンク）を必ず守る。

**qpdf parity の根拠（実測 11.9.0、本実装の truth source）:**
- document-level（outlines, names）は primary input からのみ継承（公式ヘルプ "taken from in.pdf"）。
- AcroForm field 名衝突リネーム: `name`+`name` → `name`, `name+1`；3way → `name`, `name+1`, `name+2`；候補名既存（`name`+`name`+`name+1`）→ `name`, `name+1`, `name+1+1`。
- フォーム subset: 非選択ページの widget field は出力 `/Fields` から drop、`/AcroForm` 本体は primary 由来で保持。

```rust
pub fn merge_documents<R: Read + Seek>(
    inputs: &mut [MergeInput<'_, R>],
) -> Result<Pdf<Cursor<Vec<u8>>>> {
    if inputs.is_empty() {
        return Err(Error::Unsupported("merge requires at least one input".into()));
    }
    let mut target = Pdf::open_mem_owned(minimal_target_bytes())?;
    let pages_root = target_pages_root(&mut target)?;

    // 出力順の (入力 index, コピー後ページ ref) を蓄積。
    let mut output_pages: Vec<ObjectRef> = Vec::new();
    // 全新ページ ref（keep 集合）— annotation neutralize と disjointness 検証に使う。
    let mut all_new_pages: BTreeSet<ObjectRef> = BTreeSet::new();

    for input in inputs.iter_mut() {
        let all = page_refs(input.source)?;
        // 選択 src ページ ref（範囲検証、重複可）。
        let selected = resolve_selected(&all, &input.pages)?;
        // unique 順で closure union → 1 copy_objects → 共有 dedup。
        let unique = unique_first_occurrence(&selected);
        // inherited attrs を source から materialize（extract と同方式）。
        // closure union。
        let mut closure = BTreeSet::new();
        for &s in &unique { closure.extend(page_object_closure(input.source, s)?); }

        let surviving_before: BTreeSet<ObjectRef> =
            target.object_refs().into_iter().collect();
        let map = copy_objects(input.source, &mut target, &closure)?;
        // disjointness invariant（Task 2 で debug_assert 追加）。

        // inherited materialize + reparent（extract の leaf 処理を関数化して共有）。
        // selection 順に /Kids へ。重複は shallow clone（extract と同方式）。
        // map[&s] → コピー後 first occurrence。
        // all_new_pages に追加。
        // output_pages に push。
    }

    // /Pages root を output_pages で構築、/Count 設定。
    // （dest 処理は Task 3/4。Task 1 では cross-page dest 無しの fixture のみ。）
    sweep_unreachable_objects(&mut target)?;
    Ok(target)
}
```

注意（review-patterns）:
- `input.pages` の各 index は範囲検証（`as usize` 不要だが out-of-range はエラー）。
- inherited attrs / leaf reparent / 重複 shallow clone は `extract_pages` と同一ロジック。
  実装時に `page_extract.rs:126-309` を参照し、leaf 処理を `pub(crate)` ヘルパーに
  切り出して両者で共有する（DRY）。切り出しが大きければ Task 1 を「共通ヘルパー抽出」
  サブステップに分割可。
- `map[&s]` は `map.get(&s).ok_or(Error::Missing(...))` で panic 回避。
- **Task 1 では `neutralize_absent_dests` を呼ばない**（merge は除去ページ null
  プレースホルダ機構を Task 3/4 で使う）。`all_new_pages`（keep 集合）は disjointness
  検証と Task 3/4 の判定に使うので蓄積は残す。

**Step 2: 実装**

1. **リネームヘルパー**（純関数、単体テスト可）:

```rust
/// qpdf `--pages` の field 名衝突解決: base が未使用ならそのまま、
/// 衝突するなら `base+1`, `base+2`, ... の最初の未使用名。
pub(crate) fn unique_field_name(base: &[u8], used: &BTreeSet<Vec<u8>>) -> Vec<u8> {
    if !used.contains(base) { return base.to_vec(); }
    for n in 1u32.. {
        let mut cand = base.to_vec();
        cand.extend_from_slice(format!("+{n}").as_bytes());
        if !used.contains(&cand) { return cand; }
    }
    unreachable!("u32 space exhausted")
}
```

`#[test]` で base 一致・1衝突・3連番・候補名既存（`name+1` 既存→`name+1+1`）を直接検証。

2. **マージ本体**: 各入力のコピー済みページ widget から到達する top-level field を
   primary の `/AcroForm /Fields` に追加。`AcroFormDocumentHelper::copy_fields_from`
   は「source の全 field tree を append」する設計なので、merge では:
   - primary 自身の field は primary copy で既に target に入っている（document-level
     closure に `/AcroForm` を含めれば）。primary の既存 top-level field 名を `used` の
     初期集合にする。
   - secondary 各々について、コピー済みページの widget が参照する top-level field
     （`/Parent` を辿った根）を集め、`/T` を `unique_field_name` でリネームしてから
     `/Fields` に追加。`/DR` フォント衝突は `copy_fields_from` 既存ロジックを流用。
   - **実装判断**: `copy_fields_from` を拡張して「衝突リネーム付き」モードにするか、
     page_merge に薄い専用ロジックを書く。secondary は既に target にページごとコピー
     済みなので、`copy_fields_from`（source から再コピー）とは経路が違う。
     → page_merge 側で「コピー済み widget → field root → `/T` リネーム → `/Fields` 追加」
     を実装し、`AcroFormDocumentHelper` の `ensure_acroform_ref` / `top_level_fields` を
     `pub(crate)` で借りる方が素直。実装時に両経路のコスト比較で確定。
